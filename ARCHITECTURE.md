# Project Hail Mary - Architecture

## Overview
AI-powered document translator built with Flutter. Supports multiple AI providers,
translation profiles, project memory, and advanced quality pipelines.

## Tech Stack
- **Framework**: Flutter 3.44+
- **Language**: Dart 3.12+
- **State Management**: Riverpod (StateNotifier + Provider)
- **Navigation**: GoRouter with StatefulShellRoute
- **Persistence**: Hive (9 boxes)
- **Entities**: Freezed + JSON serialization (14 files, needs build_runner)
- **Networking**: Dio with retry interceptor

## Storage (Hive Boxes)

| Box | Content |
|-----|---------|
| providers | AI provider configurations |
| settings | App settings key-value pairs |
| glossaries | Translation glossaries + entries |
| jobs | Translation job history |
| documents | Document metadata |
| projects | Translation projects + entities |

## Services

| Service | Purpose |
|---------|---------|
| ProfileEngine | Enhanced translation profiles with full prompt engineering |
| ConsistencyEngine | Character/term extraction + translation memory |

| TranslationCacheService | Content-hash-based translation caching |

| CostOptimizer | Provider selection by strategy (single source of truth for pricing) |

| PluginManager | Extensible plugin lifecycle |

## Code Generation

15 files require `build_runner` to generate ~28 output files:

- 14 entity files with `@freezed` annotations -> `.freezed.dart` + `.g.dart`
- 1 router file with `@riverpod` annotation -> `.g.dart`

```bash
dart run build_runner build --delete-conflicting-outputs
```
